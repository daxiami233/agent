# Agent 核心接口文档

本文档只描述 `agent_runtime` 核心对外接口，不包含 Web UI 的 HTTP 接口。Web 层应该通过 `Agent` 门面调用核心能力，不直接依赖 `AgentLoop` 内部实现。

## 推荐导入

最常用的 SDK 入口都从 `agent_runtime` 顶层导入：

```python
from agent_runtime import (
    Agent,
    AgentResponse,
    AgentRuntimeConfig,
    DefaultPermissionPolicy,
    PermissionDecision,
    PermissionPolicyProtocol,
    PermissionProfile,
    PermissionRequest,
    Provider,
    SkillManifest,
    ToolSpec,
    create_agent,
    tool,
)
```

## 快速开始

```python
from agent_runtime import AgentRuntimeConfig, create_agent

agent = create_agent(
    config=AgentRuntimeConfig(
        model="deepseek-v4-pro",
        api_key="your-key",
        base_url="https://example.com/v1",
        memory_backend="sqlite",
        context_window_tokens=32000,
    )
)

response = agent.run("梳理一下当前项目", conversation_id="conv-1")
print(response.text)
```

流式调用：

```python
for event in agent.stream("列出当前目录文件", conversation_id="conv-1"):
    if event.type == "assistant_delta":
        print(event.payload["text"], end="")
    elif event.type == "tool_call_start":
        print("tool:", event.payload["name"])
```

## create_agent

`create_agent(...) -> Agent` 是推荐的构造入口。它会组装 Provider、Memory、Context、Tool、Skill、MCP、Permission 等核心组件。

常用参数：

| 参数 | 类型 | 说明 |
|---|---|---|
| `model` | `str \| None` | 直接指定模型名，优先级高于 config/env |
| `api_key` | `str \| None` | 模型 API Key |
| `base_url` | `str \| None` | OpenAI-compatible API 地址 |
| `config` | `AgentRuntimeConfig \| None` | 运行时配置 |
| `provider` | `Provider \| None` | 自定义模型 Provider |
| `tools` | `list[ToolSpec] \| None` | 额外注册的业务工具 |
| `skills` | `list[SkillManifest] \| None` | 额外注册的技能 |
| `system_prompt` | `str \| None` | 基础系统提示词模板 |
| `memory_store` | `MemoryStoreProtocol \| None` | 自定义短期对话存储 |
| `long_term_memory` | `LongTermMemoryProtocol \| None` | 自定义长期记忆 |
| `token_counter` | `TokenCounterProtocol \| None` | 自定义 token 计数器 |
| `compressor` | `ContextCompressor \| None` | 自定义上下文压缩器 |
| `mcp_host` | `MCPClientHost \| None` | MCP 工具宿主 |
| `permission_policy` | `PermissionPolicyProtocol \| None` | 自定义权限策略 |
| `working_directory` | `Path \| str \| None` | 内置 shell/patch 工具默认工作目录 |

示例：只注册业务工具，不启用 shell：

```python
from agent_runtime import AgentRuntimeConfig, create_agent, tool

@tool(effects=["read"])
def query_order(order_id: str) -> dict:
    """Query order detail."""
    return {"order_id": order_id, "status": "paid"}

agent = create_agent(
    config=AgentRuntimeConfig(
        include_shell_tool=False,
        include_apply_patch_tool=False,
    ),
    tools=[query_order],
)
```

## Agent

`Agent` 是核心对外门面。应用层、Web 层、业务服务层都应该优先调用 `Agent`，不要直接调用 `AgentLoop`。

### stream

```python
Agent.stream(
    message: str,
    *,
    conversation_id: str | None = None,
    reasoning_enabled: bool = True,
    permission_profile: PermissionProfile | None = None,
    is_cancelled: Callable[[], bool] | None = None,
) -> Iterator[AgentEvent]
```

作用：

- 写入本轮用户消息
- 构建上下文
- 调用模型
- 执行工具或触发权限确认
- 返回标准化事件流

示例：

```python
events = agent.stream(
    "删除 hello.txt",
    conversation_id="conv-1",
    permission_profile="workspace",
)

for event in events:
    print(event.type, event.payload)
```

### run

```python
Agent.run(
    message: str,
    *,
    conversation_id: str | None = None,
    reasoning_enabled: bool = True,
    permission_profile: PermissionProfile | None = None,
    is_cancelled: Callable[[], bool] | None = None,
) -> AgentResponse
```

非流式调用，会收集完整事件并返回 `AgentResponse`。

```python
response = agent.run("当前文件夹是什么", conversation_id="conv-1")

print(response.text)
print(response.reasoning)
print(response.steps)
print(response.usage)
```

### continue_conversation

```python
Agent.continue_conversation(
    conversation_id: str,
    *,
    reasoning_enabled: bool = True,
    permission_profile: PermissionProfile | None = None,
    is_cancelled: Callable[[], bool] | None = None,
) -> Iterator[AgentEvent]
```

用于继续一个已经写入用户消息的对话。正常用户输入不要调用这个接口，应该调用 `stream()` 或 `run()`。

典型使用场景：

- 兼容旧的确认流程
- 外部系统已经自行写入了 user turn
- 需要在不追加新用户消息的情况下继续闭环

### resume_permission

```python
Agent.resume_permission(
    permission_id: str,
    *,
    approved: bool,
    reasoning_enabled: bool = True,
    is_cancelled: Callable[[], bool] | None = None,
) -> Iterator[AgentEvent]
```

用于恢复被权限确认暂停的工具调用。

```python
first_events = list(agent.stream("删除 hello.txt", conversation_id="conv-1"))

permission_event = next(e for e in first_events if e.type == "permission_request")
permission_id = permission_event.payload["permission_id"]

approved_events = agent.resume_permission(permission_id, approved=True)
for event in approved_events:
    print(event.type, event.payload)
```

拒绝时：

```python
events = agent.resume_permission(permission_id, approved=False)
```

核心会把拒绝写成一条 synthetic tool result，避免模型后续不知道用户已经拒绝。

### 其他方法

| 方法 | 说明 |
|---|---|
| `add_tool(tool)` | 运行时追加一个工具 |
| `add_skill(skill)` | 运行时追加一个技能，并刷新后续系统提示词 |
| `new_conversation_id()` | 生成新的 conversation id |
| `clear_conversation(conversation_id)` | 清空指定对话消息 |
| `list_conversations()` | 列出短期记忆中的对话 |

## AgentResponse

`Agent.run()` 返回：

| 字段 | 类型 | 说明 |
|---|---|---|
| `text` | `str` | 最终 assistant 回复 |
| `conversation_id` | `str` | 对话 ID |
| `reasoning` | `str` | 模型推理文本，取决于 Provider 是否返回 |
| `steps` | `list[dict]` | 工具调用步骤 |
| `usage` | `dict` | token 用量 |
| `notices` | `list[dict]` | 上下文压缩、错误等提示 |
| `events` | `list[AgentEvent]` | 完整事件列表 |

## AgentEvent

`Agent.stream()`、`Agent.continue_conversation()`、`Agent.resume_permission()` 都返回 `AgentEvent`。

常见事件：

| 事件类型 | 说明 |
|---|---|
| `reasoning_delta` | 推理文本增量 |
| `assistant_start` | assistant 回复开始 |
| `assistant_delta` | assistant 回复文本增量 |
| `tool_call_start` | 工具开始执行 |
| `tool_call_result` | 工具执行完成、失败或被拒绝 |
| `permission_request` | 工具调用需要用户确认 |
| `permission_denied` | 权限策略直接拒绝 |
| `notice` | 系统提示、错误、上下文压缩状态 |
| `usage` | token 用量 |

工具事件示例：

```python
if event.type == "tool_call_result":
    payload = event.payload
    print(payload["name"], payload["status"])
```

工具结果状态：

| 状态 | 说明 |
|---|---|
| `running` | Web 前端内部展示状态，核心通常发 `tool_call_start` |
| `completed` | 工具执行成功 |
| `error` | 工具执行失败 |
| `denied` | 用户拒绝或权限拒绝形成的工具结果 |

## AgentRuntimeConfig

运行时配置类，既可以直接构造，也可以通过 `.env` 读取：

```python
from agent_runtime import AgentRuntimeConfig

config = AgentRuntimeConfig.from_env()
```

常用字段：

| 字段 | 默认值 | 说明 |
|---|---:|---|
| `model` | `None` | 模型名称 |
| `api_key` | `None` | API Key |
| `base_url` | `None` | OpenAI-compatible base URL |
| `api_mode` | `None` | `auto`、`responses`、`chat_completions` |
| `context_window_tokens` | `32000` | 模型总上下文窗口 |
| `reserved_output_tokens` | `4000` | 输出预留 token |
| `safety_margin_tokens` | `1000` | 安全余量 |
| `compact_threshold_ratio` | `0.8` | 触发压缩的输入预算比例 |
| `recent_turns` | `6` | turn 压缩时保留的近期用户轮数 |
| `raw_keep_ratio` | `0.6` | 近期 turn 内继续压缩时保留的原文比例 |
| `memory_backend` | `sqlite` | `sqlite` 或 `memory` |
| `include_memory_tools` | `True` | 是否注册记忆工具 |
| `include_skill_tools` | `True` | 是否注册 skill 读取工具 |
| `include_shell_tool` | `True` | 是否注册 shell 工具 |
| `include_apply_patch_tool` | `True` | 是否注册 apply_patch 工具 |
| `tool_permission_profile` | `workspace` | 默认权限档位 |
| `shell_timeout_seconds` | `30` | shell 超时 |
| `shell_max_output_chars` | `20000` | shell 直接返回给模型的最大输出字符数 |

对应环境变量见 `README.md` 的 Configuration 表。

## Provider 接口

自定义模型 Provider 需要实现 `Provider`：

```python
from collections.abc import Iterator
from agent_runtime.providers import (
    ModelConfig,
    ModelResponse,
    ModelStreamEvent,
    Provider,
)

class MyProvider(Provider):
    model = "my-model"

    def generate(
        self,
        input: str | list[dict],
        *,
        instructions: str | None = None,
        tools: list[dict] | None = None,
        model_config: ModelConfig | None = None,
    ) -> ModelResponse:
        return ModelResponse(content="hello", finish_reason="stop")

    def stream(
        self,
        input: str | list[dict],
        *,
        instructions: str | None = None,
        tools: list[dict] | None = None,
        model_config: ModelConfig | None = None,
    ) -> Iterator[ModelStreamEvent]:
        yield ModelStreamEvent(type="content_delta", delta="hello")
        yield ModelStreamEvent(
            type="finish",
            response=ModelResponse(content=None, finish_reason="stop"),
        )
```

Provider 相关数据结构：

| 类型 | 说明 |
|---|---|
| `ModelConfig` | 单次模型请求配置 |
| `RetryConfig` | 重试配置 |
| `ToolCall` | 模型请求工具调用的标准结构 |
| `ModelResponse` | 非流式完整响应 |
| `ModelStreamEvent` | 流式事件 |
| `ProviderError` | Provider 异常 |
| `OpenAIProvider` | 内置 OpenAI-compatible Provider |

## Tool 接口

### 使用装饰器创建工具

```python
from agent_runtime import tool

@tool(
    description="Read one business entity.",
    effects=["read"],
)
def get_customer(customer_id: str) -> dict:
    return {"customer_id": customer_id, "name": "Alice"}
```

### 手动创建 ToolSpec

```python
from agent_runtime import ToolSpec

def handler(arguments: dict) -> dict:
    return {"echo": arguments["text"]}

echo = ToolSpec(
    name="echo",
    description="Echo text.",
    input_schema={
        "type": "object",
        "properties": {"text": {"type": "string"}},
        "required": ["text"],
        "additionalProperties": False,
    },
    handler=handler,
    effects=["read"],
)
```

`ToolSpec` 字段：

| 字段 | 说明 |
|---|---|
| `name` | 工具名，会暴露给模型 |
| `description` | 工具说明 |
| `input_schema` | JSON Schema 参数定义 |
| `handler` | Python 调用函数 |
| `capabilities` | 能力标签，例如 `memory` |
| `risk_level` | `auto`、`confirm`、`blocked` |
| `effects` | `read`、`write`、`execute`、`network`、`destructive` |

### ToolRegistry

高级用法可以直接使用 `ToolRegistry`：

```python
from agent_runtime.tools import ToolRegistry

registry = ToolRegistry([echo])
registry.execute("echo", {"text": "hi"})
```

### 内置工具

| 工具 | 说明 | effect |
|---|---|---|
| `memory_read` | 读取长期记忆 | `read` |
| `memory_search` | 搜索长期记忆 | `read` |
| `memory_append` | 追加长期记忆 | `write` |
| `memory_replace` | 覆盖长期记忆 | `write` |
| `skill_read` | 读取 skill 元数据和说明 | `read` |
| `skill_read_resource` | 读取 skill 目录内资源文件 | `read` |
| `list_skills` | 列出已注册 skills | `read` |
| `shell_command` | 在工作目录执行 shell | `execute` |
| `apply_patch` | 使用 patch 修改文件 | `write` |

## Permission 接口

权限档位：

| 档位 | 说明 |
|---|---|
| `read_only` | 只允许安全读取，拒绝写入/执行/网络/破坏性操作 |
| `workspace` | 允许常见读取、测试、构建；写入和危险 shell 需要确认 |
| `full_access` | 尽量直接允许，但 blocked 工具仍拒绝 |

默认策略：

```python
from agent_runtime import DefaultPermissionPolicy

policy = DefaultPermissionPolicy(profile="workspace")
```

业务自定义策略：

```python
from agent_runtime import PermissionDecision, PermissionRequest
from agent_runtime.permissions import PermissionPolicyProtocol

class BusinessPermissionPolicy:
    def evaluate(self, request: PermissionRequest) -> PermissionDecision:
        if request.tool_name == "delete_order":
            return PermissionDecision("confirm", "Deleting orders requires approval.")
        return PermissionDecision("allow", "Allowed by business policy.")

agent = create_agent(permission_policy=BusinessPermissionPolicy())
```

`PermissionRequest` 字段：

| 字段 | 说明 |
|---|---|
| `id` | 工具调用 ID |
| `conversation_id` | 当前对话 ID |
| `tool_name` | 工具名 |
| `arguments` | 工具参数 |
| `risk_level` | 工具风险等级 |
| `effects` | 工具影响 |
| `capabilities` | 工具能力标签 |
| `profile` | 本轮权限档位 |

`PermissionDecision.action` 可以是：

| action | 行为 |
|---|---|
| `allow` | 允许执行 |
| `confirm` | 暂停并返回 `permission_request` |
| `deny` | 直接拒绝 |

## Skill 接口

Skill 是模型可用能力的轻量元数据，完整说明通常放在 `SKILL.md`。

```python
from agent_runtime import SkillManifest

skill = SkillManifest(
    name="order_skill",
    description="Handle order query and refund workflows.",
    triggers=["order", "refund"],
    context_files=["refund_policy.md"],
    required_tools=["query_order", "refund_order"],
)

agent.add_skill(skill)
```

加载本地 skill：

```python
from agent_runtime.skills import load_skill, load_skills

one = load_skill("./skills/order_skill")
many = load_skills("./skills")
```

`SkillRegistry` 可用于高级场景：

```python
from agent_runtime.skills import SkillRegistry

registry = SkillRegistry([skill])
prompt = registry.apply_to_system_prompt()
```

## Memory 接口

短期对话存储：

| 类型 | 说明 |
|---|---|
| `MemoryStore` | 兼容别名，当前指向 SQLite 存储 |
| `SQLiteMemoryStore` | SQLite 持久化对话存储 |
| `InMemoryMemoryStore` | 进程内存存储，适合测试 |
| `MemoryStoreProtocol` | 自定义短期存储协议 |
| `ConversationRecord` | 对话元数据 |
| `StoredMessage` | 对话消息 |

长期记忆：

| 类型 | 说明 |
|---|---|
| `LongTermMemory` | 进程内长期记忆 |
| `SQLiteLongTermMemory` | SQLite 持久化长期记忆 |
| `LongTermMemoryProtocol` | 自定义长期记忆协议 |
| `MemoryRecord` | 长期记忆记录 |

示例：

```python
from agent_runtime.memory import InMemoryMemoryStore, LongTermMemory

agent = create_agent(
    memory_store=InMemoryMemoryStore(),
    long_term_memory=LongTermMemory(),
)
```

## Context 接口

普通业务集成通常不需要直接使用 Context 接口。高级场景可注入自定义 token counter 或 compressor。

主要类型：

| 类型 | 说明 |
|---|---|
| `ContextEngine` | 构建模型输入、压缩上下文、读写对话消息 |
| `ContextMessage` | 核心内部消息结构 |
| `ContextOverflowError` | 压缩后仍超预算时抛出 |
| `ContextBudget` | 上下文预算结构 |
| `TokenCounter` | 默认 token 计数器 |
| `TokenCounterProtocol` | 自定义 token 计数协议 |
| `ContextCompressor` | 压缩器接口 |
| `ModelContextCompressor` | 使用模型生成摘要的压缩器 |
| `CompressionResult` | 压缩结果 |

自定义压缩器：

```python
from agent_runtime.context import CompressionResult, ContextCompressor

class FixedCompressor(ContextCompressor):
    def compress(self, *, conversation_id, messages, target_tokens):
        return CompressionResult(
            summary="Earlier conversation was summarized.",
            compressed=True,
        )

agent = create_agent(compressor=FixedCompressor())
```

## MCP 接口

`MCPClientHost` 用于把 MCP 工具注册到核心工具列表。当前核心接口是轻量宿主，后续可扩展 MCP server 生命周期管理。

```python
from agent_runtime.mcp import MCPClientHost

mcp_host = MCPClientHost()
agent = create_agent(mcp_host=mcp_host)
```

## 日志接口

```python
from agent_runtime.runtime import runtime_log, runtime_log_path

runtime_log("custom_event", {"value": 1})

```

## 推荐集成方式

Web 或业务系统应该遵守以下边界：

1. 用户输入：调用 `Agent.stream()` 或 `Agent.run()`。
2. 权限确认：调用 `Agent.resume_permission()`。
3. 已写入 user turn 后继续：调用 `Agent.continue_conversation()`。
4. 工具扩展：传入 `tools=[...]` 或调用 `agent.add_tool(...)`。
5. 业务权限：注入 `permission_policy=...`。
6. 业务流程：优先沉淀为 skill + tool，不要修改核心 loop。

不推荐：

```python
agent.loop.run(...)
agent.loop.resume_permission(...)
```

这些属于核心内部实现细节，Web 层和业务层不应该直接依赖。

## 最小业务接入示例

```python
from agent_runtime import (
    AgentRuntimeConfig,
    PermissionDecision,
    PermissionRequest,
    create_agent,
    tool,
)

@tool(effects=["read"])
def query_order(order_id: str) -> dict:
    """Query one order."""
    return {"order_id": order_id, "status": "paid"}

@tool(effects=["write"], risk_level="confirm")
def refund_order(order_id: str, reason: str) -> dict:
    """Refund one order."""
    return {"order_id": order_id, "refunded": True, "reason": reason}

class OrderPermissionPolicy:
    def evaluate(self, request: PermissionRequest) -> PermissionDecision:
        if "write" in request.effects:
            return PermissionDecision("confirm", "Order writes require approval.")
        return PermissionDecision("allow", "Read operation.")

agent = create_agent(
    config=AgentRuntimeConfig(
        include_shell_tool=False,
        include_apply_patch_tool=False,
    ),
    tools=[query_order, refund_order],
    permission_policy=OrderPermissionPolicy(),
)

for event in agent.stream("查询订单 A001，如果已付款就退款", conversation_id="order-1"):
    if event.type == "permission_request":
        permission_id = event.payload["permission_id"]
        break

for event in agent.resume_permission(permission_id, approved=True):
    print(event.type, event.payload)
```
